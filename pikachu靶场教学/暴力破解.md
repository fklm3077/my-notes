# 4.token防爆破
## 1.token的基本知识
### <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一、Token基础定义</font>
**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">Token = 令牌 / 临时凭证</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">简单说：</font>

+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">服务端生成一串</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">随机字符串</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">发给前端（页面 / 浏览器）</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">前端每次发请求</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">必须带上这串 Token</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">服务端</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">验证 Token 有效才处理请求</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">用过</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一次</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">就失效 / 定时失效</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一句话总结：Token 就是服务端给客户端发的 “</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一次性临时</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">门票”。</font>

### <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">二、Token作用</font>
1. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">防止暴力猜密码：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">每次请求必须带新 Token，暴力脚本</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">无法自动连续发包</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">。</font>
2. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">防止重复提交：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">同一个 Token 只能用一次。</font>
3. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">防止 CSRF 跨站请求伪造：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">攻击者拿不到你的随机 Token，无法伪造请求。</font>

### <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">三、Token防暴破的原理</font>
<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">Pikachu 防爆破模块的工作流程：</font>

1. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">打开登录页面，服务端生成一个 </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">随机 Token</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">，藏在表单里，查看网页源代码</font>

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/60942548/1777553914073-e871262b-edf0-4c08-ad42-2f58caec83e2.png)

2. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">你输入账号密码提交</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">username=test</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">password=test</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">token=abcd1234</font>
3. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">服务端做两件事</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">验证 Token 是否存在、是否有效</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">验证完 </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">立即销毁</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">这个 Token（一次性）</font>
4. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">返回新页面 </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">→ 生成新 Token</font>**

### <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">四、为什么能防爆破</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">暴力破解工具（Burp）默认只会</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">重放同一个包</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">但 Token </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一次性</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">有效</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">重放 → Token 失效 → 登录失败</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">必须</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">每次先获取新 Token，再发包</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">才能成功</font>
